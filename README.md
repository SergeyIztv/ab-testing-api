AB Testing API

Backend-сервис для управления A/B экспериментами, разработанный на Django.

Стек

Python 3
Django
Django REST Framework
SQlite

Описание

Сервис используется для проведения A/B тестирования в мобильных или веб-приложениях.

Приложение получает список активных экспериментов и для каждого устройства детерминированно выбирает вариант.

Это означает, что один и тот же пользователь (Device-Token) всегда получает одинаковые значения в рамках одного эксперимента.

Если эксперимент был создан после первого обращения пользователя, он ему не показывается.

Пример использования
1. Запрос активных экспериментов

GET /api/v1/experiments/

Заголовок запроса:

Device-Token: 123e4567-e89b-12d3-a456-426614174000

2. Пример ответа сервера
{
  "experiments": [
    {
      "key": "button_color",
      "value": "#FF0000"
    },
    {
      "key": "price",
      "value": "10"
    }
  ]
}
3. Пример интерпретации на клиенте
Эксперимент button_color управляет цветом кнопки "Купить"
Эксперимент price управляет отображаемой ценой товара

Пример логики на клиенте:

Если key = button_color, применить значение как цвет UI элемента
Если key = price, показать соответствующую цену пользователю

Примеры экспериментов

button_color
Варианты: #FF0000, #00FF00, #0000FF
Равномерное распределение между пользователями

price
Варианты: 10 (75%), 20 (10%), 50 (5%), 5 (10%)

API

GET /api/v1/experiments/

Описание: возвращает список активных экспериментов для конкретного устройства

Заголовки:
Device-Token — уникальный идентификатор устройства

Статистика

/admin/stats/ — интерфейс для просмотра распределения пользователей по вариантам экспериментов

Запуск проекта
git clone git@github.com:SergeyIztov/ab-testing-api.git
cd ab-testing-api

python -m venv venv
source venv/bin/activate

pip install -r requirements.txt

python manage.py migrate
python manage.py runserver
